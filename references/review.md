# Review / Codeview

用于任务开发完成或准备上线前，对当前任务改动做代码审查、项目规则审查、对抗式审查，并收敛测试代码。

显式命令：

```text
/task-workflow review [目标1] [目标2] [...]
/task-workflow codeview [目标1] [目标2] [...]
```

如果用户说“代码审查”“检查代码改动”“收敛测试代码”“一次性测试可以去掉”，按 `review` 流程处理。`view` 只有在上下文明确是代码审查时才按 `review` 理解，否则不要和 Portal / Status 混用。

目标表达方式与 publish / sync 相同：
- 默认不指定目标时，审查当前任务下全部绑定仓库
- 指定目标时，继续使用和 publish 一样的自然语言目标识别方式，例如 `后端`、`前端`、`手机前端`、`PC前端`

## 默认流程

1. 先按 Load 协议恢复 `meta.yaml -> index.md -> plan.md -> progress.md`
2. 确认当前任务目标、当前阶段、绑定仓库、记录分支和当前工作区状态
3. 对每个目标仓库先 `fetch origin --prune`，再用远程默认分支对比当前任务分支，优先使用 `origin/master...HEAD` 或实际远程默认分支
4. 先看文件清单、diff stat 和任务相关路径，再决定是否需要阅读全文；长生命周期或大 diff 任务要先收敛 review scope
5. 查找并读取项目级 review 规则；有则执行，无则说明未发现项目级 review 规则
6. 做通用 code review
7. 做对抗式审查
8. 做测试代码收敛审查
9. 输出 findings、风险、测试收敛建议和验证建议

## 项目级 Review 规则

- 优先在目标仓库内查找 `AGENTS.md`、`CLAUDE.md`、`docs/AI_RULES.md`、`docs/review-prompt.md`、`CONTRIBUTING.md`、`CODE_REVIEW.md`
- 如果多个仓库都有规则，按仓库分别应用，不要把一个仓库的规则无条件套到另一个仓库
- 如果没有找到项目级规则，不要虚构规则，只执行通用 review 和本 skill 的审查要求

## 通用 Code Review 关注点

- 真实 bug、边界条件、异常路径、兼容性、数据一致性、前后端契约、权限、安全、性能、可维护性
- 后端任务要重点看表/字段/枚举默认值、旧数据兼容、查询范围、索引和上线数据变更
- 前端任务要重点看入口、状态流转、筛选/详情/弹窗联动、权限态、空态、错误态、复用既有交互和组件
- 多仓任务要重点看前后端字段、枚举、默认值、错误处理和发布顺序不一致时的兼容

## 对抗式审查

- 主动站在“哪里会失败、哪里会被绕过、哪里和用户预期不一致”的角度审查
- 必查场景：无权限、跨店铺/跨加工厂、历史数据、空数据、异常数据、并发/重复提交、部分仓库未同步上线、接口返回缺字段或旧字段
- 对不确定问题只标为风险或待确认，不把产品口径猜测当作明确 bug
- 明确 bug 和缺陷可以给出修复建议；是否直接修复取决于用户是否授权

## 测试代码收敛

- 默认把测试改动作为独立 review pass，不把测试文件数量多当作天然合理
- 优先保留能防真实回归的行为测试、边界测试、权限/数据范围测试、历史 bug 回归测试
- 优先收敛或删除一次性测试、临时验证脚本、只验证实现细节的断言、视觉偏好断言、重复覆盖同一分支的测试
- 如果测试没有接入默认脚本、CI 或稳定运行路径，默认按一次性测试处理，除非它有明确长期回归价值
- 表结构、索引、DDL、ORM 字段、模型 import 注册、AST 形状等测试，默认归为方案或上线前校验，不作为长期单测保留
- 结构/索引校验应看字段/枚举/默认值、索引目的、查询入口、数据量、SQL 条件、必要的 `EXPLAIN` 和新旧数据兼容，而不是靠长期测试文件兜底
- 因 AI 跑测试引入的无关 mock/fake 依赖、短信/MQ/云服务/通用参数类绕行测试，如果和本任务业务无关，按测试污染处理，建议删除或回退
- 测试收敛结论建议分成四类输出：长期保留、合并精简、移到方案/上线前校验、删除无关污染
- 收敛测试代码时，先说明保留/删除判断；只有用户授权“明确问题直接修”“收敛测试代码”时才修改测试文件

## 修复边界

- 默认只做审查和输出结论，不直接改业务代码或测试代码
- 如果用户明确说“明确问题直接修”“测试代码可以收敛/删除”，可以处理明确缺陷和低价值测试
- 不确定问题、产品口径问题、较大重构、范围扩大问题，先列为待确认，不自行推进
